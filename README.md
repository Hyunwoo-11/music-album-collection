# 음악 앨범 컬렉션 웹 애플리케이션 최종 보고서

## 1. 실험의 목적과 범위

본 프로젝트의 목적은 Spotify API를 활용하여 앨범을 검색하고, 이를 사용자의 컬렉션으로 관리할 수 있는 웹 애플리케이션을 구현하는 것이다. 
주 사용자는 LP나 CD를 수집하는 사람을 대상으로 하고 있다.

포함 내용:
- 앨범 검색 기능 (Spotify API 활용)
- 컬렉션 추가, 조회, 수정, 삭제 기능 (CRUD)
- 프론트엔드와 백엔드 연동

불포함 내용:
- 사용자 로그인 기능
- 추천 시스템
- 대규모 데이터 처리

---

## 2. 분석

본 시스템은 다음과 같은 주요 기능으로 구성된다.

- 앨범 검색
- 검색 결과 확인
- 컬렉션 추가
- 컬렉션 조회
- 상태 변경 (WISH / OWNED)
- 컬렉션 삭제

사용자는 앨범 검색을 통해 결과를 확인하고, 원하는 앨범을 컬렉션에 추가할 수 있으며, 이후 컬렉션을 관리할 수 있다.

## 3. 설계

### (1) 시스템 구조

- 프론트엔드: React
- 백엔드: Node.js (Express)
- 데이터베이스: MySQL


### (2) 데이터 구조

- albums 테이블
  - id (Primary Key)
  - spotify_id
  - title
  - artist
  - release_date
  - cover_image
  - status (WISH / OWNED)

### (3) API 구현

본 시스템은 REST API 기반으로 구현되었으며, 다음과 같은 주요 엔드포인트를 사용하였다.

| Method | URL | 설명 |
|--------|-----|------|
| GET | /api/search | 앨범 검색 (Spotify API 연동) |
| POST | /api/collection | 앨범 컬렉션 추가 |
| GET | /api/collection | 컬렉션 조회 |
| PUT | /api/collection/:id | 앨범 상태 변경 |
| DELETE | /api/collection/:id | 컬렉션 삭제 |

프론트엔드에서는 axios를 사용하여 해당 API를 호출하고, 서버에서는 Express를 통해 요청을 처리한 후 MySQL 데이터베이스와 연동하여 데이터를 관리하였다.

### (4) 처리 흐름

1. 사용자가 검색어 입력
2. 서버가 Spotify API를 호출하여 결과 반환
3. 사용자가 컬렉션에 추가 요청
4. 서버가 DB에 저장
5. 조회, 수정, 삭제 기능 수행

## 4. 구현

### (1) 개발 환경
- Frontend: React (Vite)
- Backend: Node.js (Express)
- Database: MySQL
- API: Spotify Web API
- 개발 도구: Visual Studio Code, Thunder Client

### (2) 주요 구현 내용

- React의 useState를 활용한 상태 관리
- axios를 활용한 REST API 통신
- Express를 이용한 API 서버 구현
- MySQL과 연동하여 데이터 저장 및 조회 기능 구현

## 5. 실험

### (1) 테스트 방법
- Thunder Client를 이용하여 API 테스트 수행
- React 화면에서 기능 동작 확인

### (2) 테스트 시나리오

| 번호 | 기능 | 테스트 내용 | 입력값 | 결과 |
|------|------|------------|--------|-----------|
| 1 | 앨범 검색 | 검색어 입력 후 검색 | IU | IU 관련 앨범 목록이 출력됨 |
| 2 | 앨범 검색 (예외) | 빈 값으로 검색 | "" | 검색 요청이 수행되지 않음 |
| 3 | 컬렉션 추가 | 앨범을 컬렉션에 추가 | 앨범 데이터(JSON) | DB에 데이터 저장 및 성공 메시지 출력 |
| 4 | 컬렉션 조회 | 저장된 앨범 목록 조회 | 없음 | DB에 저장된 앨범 목록이 출력됨 |
| 5 | 상태 변경 | WISH → OWNED 변경 | id=1, status=OWNED | 해당 앨범 상태가 변경됨 |
| 6 | 컬렉션 삭제 | 앨범 삭제 | id=1 | 해당 앨범이 DB에서 삭제됨 |
| 7 | 컬렉션 검색 | 컬렉션 내 검색 | IU | IU 포함된 앨범만 필터링 출력 |
| 8 | API 오류 | 잘못된 요청 처리 | 잘못된 id | 오류 메시지 반환 |

### (3) 테스트 결과
- 앨범 검색 기능 정상 동작
- 컬렉션 추가, 조회, 수정, 삭제 기능 정상 동작
- 데이터가 MySQL DB에 정상적으로 저장 및 반영됨

## 6. 결론

본 프로젝트에서는 Spotify API를 활용한 앨범 검색 기능과, MySQL 데이터베이스를 이용한 컬렉션 관리 기능(CRUD)을 구현하였다.  
사용자는 앨범을 검색하고, 원하는 앨범을 컬렉션에 추가할 수 있으며, 이후 조회, 상태 변경, 삭제 기능을 통해 데이터를 관리할 수 있도록 시스템을 구성하였다.  
이를 통해 React와 Node.js를 활용한 웹 애플리케이션 개발 과정을 경험할 수 있었으며, 프론트엔드와 백엔드의 연동 구조를 이해할 수 있었다. 또한 REST API를 직접 구현하고 외부 API를 활용함으로써 실제 서비스와 유사한 구조를 구현할 수 있었고, 데이터 흐름과 처리 과정을 학습할 수 있었다.  
특히, 외부 API를 직접 활용해보고자 하는 목적에서 Spotify API를 선택하여 앨범 데이터를 활용한 기능을 구현하였으며, 이를 통해 실제 서비스에서 사용하는 데이터 처리 방식과 API 연동 과정을 경험할 수 있었다.
